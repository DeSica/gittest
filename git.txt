git config --global 设置全局变量 user.name user.email

git status	查看状态

git log	显示提交记录

git reflog	显示所有记录

HEAD^ HEAD~100	指向版本库的指针

git reset --hard	回到某个版本

git commit -m ""	提交记录

git add	添加到暂缓区

git diff a -- b	比对

git checkout -- file	工作区/暂存区 状态复原

git reset HEAD file		暂存区ADD撤销

git rm	删除文件   也是修改也需要commit

ssh-keygen -t rsa -C "youremail@example.com"	创建SHH Key 后在github上添加ssh key

git remote add origin git@github.com:michaelliao/learngit.git 关联远程仓库

git push -u origin master	将本地仓库推送到远程
由于远程库是空的，第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，
还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令为git push origin master
